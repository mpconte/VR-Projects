Profile File Format

A profile contains user-specific settings for the library. The structures are defined in detail in the ve_env.h module of the VE API Reference Documentation. The basic syntax of the file is based around a list syntax described in the ve_tlist.h module.

A profile is comprised of a number of statements. The following statements are valid:

name string
Names the profile. This should be a short unique name for the user without spaces.
fullname string
The user's full name for informational purposes.
eyedist dist
The interpupilary distance for this user in metres.
module modulename {moduledata ...}
A generic model for other per-user information. Other information is grouped into modules. Each driver or application would define its own module and look there for per-user data that is specific to that driver or application. Module data is simply a pair of a name and a value. Values are stored as strings - interpretation is up to the specific driver or application that uses the data. Each pair is on a separate line.

Example

The following is a simple example: 

        name joe
        fullname "Joe Somebody"
        eyedist 0.065
        module headtrack {
                offset {1.0 0.2 0.0}
                calibrated 1
        }
