Function: veBlueInit
void veBlueInit(void);
Initializes VE's BlueScript interpreter. This is typically called by the veInit() function.

Type: VeBlueStringProc
As an alternative to the standard BlueScript procedure interface, this VE-specific procedure type allows communication with strings. The simplified interfaces veBlueSetResult(), veBlueSetVar(), veBlueGetVar() and veBlueSetStringProc() can also be used.

When called, this procedure should always return either BS_OK or BS_ERROR to indicate respectively that execution completely successfully or an error occurred. Use the veBlueSetResult() function to specify the result of the procedure (if return value is BS_OK) or an error message (if return value is BS_ERROR).

Parameters to the procedure are passed as an array of strings. argv[0] is the name of the procedure.

typedef int (*VeBlueStringProc)(int argc, char **argv);

Function: veBlueSetResult
void veBlueSetResult(char *value);
Sets the result of a procedure call to a copy of the given string.
value - The result of the procedure call. This overrides any previously set result. If value is NULL then the result is an empty string.

Function: veBlueSetVar
void veBlueSetVar(char *name, int global, char *value);
Sets a variable in the BlueScript interpreter.
name - The name of the variable to set.
global - If this is 0, then the variable is set in the local context. Otherwise it is set in the global context.
value - The value of the variable. Passing NULL here will unset the variable.

Function: veBlueGetVar
char *veBlueGetVar(char *name, int global);
Retrieves the value of a variable from the BlueScript interpreter.
name - The name of the variable to retrieve.
global - If this is 0, then the variable is retrieved from the local context. Otherwise, it is retrieved from the global context.
Returns: A pointer to a read-only string if the variable is found, or NULL if the variable is not found.

Function: veBlueSetProc
void veBlueSetProc(char *name, VeBlueStringProc proc);
Creates a procedure in VE's BlueScript interpreter using a simplified interface. This interface uses strings instead of BlueScript objects. This makes coding simpler at the loss of some efficiency.

Function: veBlueSetExtProc
void veBlueSetExtProc(char *name, BSExtProc proc, void *cdata);
Creates a procedure in VE's BlueScript interpreter. When the procedure is called, the given process will be called. If an existing procedure already uses this name, then the existing procedure is overridden.

Use of this function assumes that you know how to use the BlueScript objects. For a simpler but less efficient interface see the veBlueSetStringProc function.

name - The name of the procedure.
proc - A BlueScript procedure function to call when the procedure is called.

Function: veBlueGetVector3
VeVector3 *veBlueGetVector3(BSObject *o);
Attempts to interpret a BlueScript object as a VeVector3 object.
o - The object to interpret.
Returns: A pointer to a read-only VeVector3 object. If the object cannot be interpreted as a valid VeVector3 object then NULL is returned and an error message is left in the interpreter's result object.

Function: veBlueSetVector3Result
void veBlueSetVector3Result(VeVector3 *v);

v - The vector that will be stored in the result.
