File: ve_alloc.h

This module provides memory allocation and freeing abstractions for use within VE. In particular, these abstractions:

  • Verify that the allocation succeeds and aborts program execution if it fails.
  • Only frees a pointer if it is not NULL.

In addition to the functions below, the macro veAllocObj() is defined. This macro takes a datatype as an argument and allocates space for an object of that type. For example: veAllocObj(VeEnv) would allocate space to hold a VeEnv object. The bytes of the object are initialized to 0 and the value of the macro is automatically cast to a pointer of the appropriate type.

Function: veAlloc
void *veAlloc(unsigned nbytes, int zero);
Allocates a region of memory. Always returns a valid range of memory or aborts program execution.
nbytes - The number of bytes to allocate. This must not be zero. If an allocation of 0 bytes is attempted, then the program will exit.
clear - If this value is zero, then all allocated bytes will be initialized to 0. If this value is not zero, then the initial value of the allocated bytes is undefined.
Returns: A pointer to a newly-allocated region of memory.

Function: veRealloc
void *veRealloc(void *ptr, unsigned nbytes);
Reallocates a previously allocated region of memory (like realloc).

Function: veFree
void veFree(void *);

Function: veDupString
char *veDupString(char *s);

Returns a duplicate of the given string. The result has been newly allocated from the heap.

s - The string to duplicate.
Returns: A duplicate of the given string.
